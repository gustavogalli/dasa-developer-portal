Payments API / Create charge

Criar cobranca com idempotencia, payload validado e headers obrigatorios.

Core do portal: endpoint, parametros, exemplos e guias tecnicos acessiveis no mesmo contexto.

Resumo tecnico

Campos e contratos que o desenvolvedor precisa bater o olho e usar.

  • EndpointPOST /v2/payments/charges
  • AutenticacaoHeader x-api-key
  • HeadersContent-Type, x-correlation-id
  • Response201 Created com objeto charge

Parametros

Estrutura de request preparada para leitura rapida.

Campo Tipo Descricao
amount number Valor total em centavos.
currency string Moeda ISO, ex.: BRL.
customer.id string Identificador do cliente parceiro.

Exemplo de request

Snippet pronto para a primeira integracao.

curl --request POST "https://api.sandbox.dasa.com/v2/payments/charges" \
  --header "Content-Type: application/json" \
  --header "x-api-key: <YOUR_API_KEY>" \
  --header "x-correlation-id: demo-001" \
  --data '{{ "{" }}
    "amount": 12990,
    "currency": "BRL",
    "customer": {{ "{" }}
      "id": "cus_21931"
    {{ "}" }}
  {{ "}" }}'

Preview da aba "Testar" dentro da pagina da API.

Exemplos de uso

  • Criar cobranca avulsa.
  • Consultar status de processamento.
  • Tratar recusas por regra de negocio.

Boas praticas

  • Use idempotencia para evitar duplicidade.
  • Envie correlation id em todos os requests.
  • Monitore status 4xx e 5xx com retries controlados.
